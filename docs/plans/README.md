# 改进计划总览

本文档包含 ACMer-Simulator 项目的完整改进方案。

## 子计划清单

| 计划 | 文件名 | 目标 | 工期 | 依赖 |
|------|--------|------|------|------|
| **A** | [plan-A.md](./plan-A.md) | 配置提取与清理 | 3天 | 无 |
| **B** | [plan-B.md](./plan-B.md) | 代码重构 | 4天 | 无 |
| **C** | [plan-C.md](./plan-C.md) | 测试覆盖 | 5天 | 无 |
| **D** | [plan-D.md](./plan-D.md) | TypeScript基础设施 | 3天 | 无 |
| **E** | [plan-E.md](./plan-E.md) | TypeScript迁移 | 10天 | D |
| **F** | [plan-F.md](./plan-F.md) | 性能优化 | 2天 | 无 |

## 执行顺序

### 第一阶段（可并行）

**A、B、C、D、F** 完全独立，可同时开始：

- **A**: 提取配置常量 → 清理魔法数字
- **B**: 拆分长函数 → 降低复杂度
- **C**: 添加单元测试 → 确保功能正确
- **D**: 配置TS → 定义类型
- **F**: 添加memo → 减少渲染

### 第二阶段

**E** 依赖 **D** 完成：

- **E1**: 迁移工具与常量
- **E2**: 迁移数据层（traits, activities）
- **E3**: 迁移数据层（contests, events）
- **E4**: 迁移游戏逻辑
- **E5**: 迁移状态管理
- **E6**: 迁移UI组件

**注意**：E1-E3 之间可并行，E4-E6 需依次完成。

## 快速启动

### 单个子计划

直接打开对应文件按步骤执行：

```bash
# 例如执行计划A
open docs/plans/plan-A.md
```

### 完整改进

按顺序执行：

1. 并行完成 A、B、C、D、F
2. 等待 D 完成后，执行 E1-E6

## 验收标准

每个子计划有独立的验收标准，详见各文件末尾。

### 整体验收

所有计划完成后：

- [ ] 配置集中管理（A完成）
- [ ] 函数复杂度达标（B完成）
- [ ] 测试通过 >80% 覆盖（C完成）
- [ ] TypeScript 零错误（D+E完成）
- [ ] 性能改善 >30%（F完成）

## 文件位置

- 配置: `src/config/gameBalance.ts`（计划A创建）
- 类型: `src/types/index.ts`（计划D创建）
- 测试: `__tests__/*.test.js`（计划C创建）
