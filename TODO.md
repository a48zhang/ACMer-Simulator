## 新增：特性系统与开局重构（替代属性分配）

目标：重构游戏开局流程，移除手动属性分配，改为“特性（Trait）选择”模式。引入“余额”概念替代原 UI 位置的行动点显示。

### 1. UI 与资源调整
- **余额（Balance）**：
  - 将原设计中左上角的“行动点”显示位置改为显示“余额”（金钱/资金）。
  - 初始余额：固定值（如 1000，待定）。
  - 用途：未来可用于购买道具、活动报名费等（预留设计）。
- **行动点（AP）**：
  - 仍保留“每月 30 点”的机制用于限制活动频率，但在 UI 只保留主面板的显示。

### 2. 开局流程重构
- **移除**：`AttributeDialog` 中的 20 点属性分配环节。
- **新增**：`TraitSelectionDialog`（特性选择对话框）。
- **规则**：
  - 玩家从列表中选择若干特性。
  - **特性分（Trait Points, TP）**：
    - 初始 TP = 10。
    - 选择 **正面特性**：扣除 TP（TP 减少）。
    - 选择 **负面特性**：增加 TP（TP 增加，相当于返还点数）。
  - **开始条件**：最终 TP >= 0 才能开始游戏。
    - 意味着玩家最多可以“透支” 10 点 TP 来换取强力开局，或者必须搭配负面特性来平衡。

### 3. 特性设计（Traits）
特性分为正面（Buff）和负面（Debuff），影响初始属性、成长率或游戏机制。

#### 数据结构示例
```json
{
  "id": "trait_noip_1",
  "name": "高中NOIP一等奖",
  "type": "positive",
  "cost": 10, // 扣除 10 TP
  "description": "你曾是信息学奥赛选手。所有专业属性+2，额外随机分配10点属性。",
  "effects": {
    "initialStats": { "math": 2, "dp": 2, "graph": 2, ... }, // 伪代码
    "randomBonus": 10
  }
}
```

#### 预设特性列表（示例）

**正面特性（Positive）**
1.  **高中NOIP一等奖**
    -   **消耗**：10 TP
    -   **效果**：所有专业属性（Math, DP, Graph, DataStructure, String, Search, Greedy, Geometry）初始值 +2。
    -   **额外效果**：游戏开始时，额外获得 10 点属性点并随机分配到各项属性中。
2.  **英语专八**
    -   **消耗**：5 TP
    -   **效果**：`english` 属性初始值直接设为 10（满级）。
3.  **数学天才**
    -   **消耗**：8 TP
    -   **效果**：`math` 和 `geometry` 初始值 +4，且后续成长速度（如有熟练度系统）提升。

**负面特性（Negative）**
1.  **精神压抑**
    -   **收益**：+5 TP（即 Cost = -5）
    -   **效果**：SAN值（`stress` 对应的耐受度）上限降低，或每月 `stress` 自然增长率增加。
2.  **独狼**
    -   **收益**：+3 TP
    -   **效果**：`teamwork` 属性锁定为 0，无法提升。
3.  **偏科**
    -   **收益**：+4 TP
    -   **效果**：`english` 属性上限降低为 5。

### 4. 实现指引
- 修改 `App.jsx` 中的 `gameState` 初始化，移除 `availablePoints` 相关逻辑。
- 新增 `TraitSelection` 组件，用于开局前展示特性列表并计算 TP。
- 在 `startGame` 时，根据所选特性计算初始属性值（应用固定加成 + 随机分配）。
- 确保 UI 左上角展示 `balance`，并处理好 AP 的显示位置。
